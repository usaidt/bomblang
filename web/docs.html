Introduction

BombLang is an esoteric programming language built around explosive metaphors. Every operation is designed to feel like you're detonating code rather than just running it. The language uses unique symbolic syntax that makes programming feel like defusing (or setting off) bombs!

Syntax Overview

BombLang uses a unique set of symbols for different operations. Here's the complete syntax reference:

Symbol Purpose Description Example
* Variable Declaration Creates a new variable or assigns a value *5 @x
@ Assignment Assigns a value to a variable *10 @myVar
& Expression Separator Separates parts of expressions *x & + & y @sum
! Function Call/Alert Calls functions or displays alerts !alert "Hello!"
~ Function Definition Defines a new function ~myFunc-param_code_
_ Block Delimiter Marks the end of code blocks ~func_code_
- Parameter Separator Separates function parameters ~add-a-b_code_
:if_ If Statement Conditional execution *x & > & 5:if_code_
:else_ Else Statement Alternative execution :else_code_
^label_ Try-Catch Block Error handling ^error_code_^error_handler_

Variables

Variables in BombLang are declared using the asterisk (*) symbol followed by a value and assignment (@):

Basic Variable Declaration

// Declare a number variable
*42 @answer

// Declare a string variable
*"Hello World" @greeting

// Declare multiple variables
*10 @x
*20 @y
*x & + & y @sum

Variable Types

BombLang supports several data types:

  • Numbers: *42 @num
  • Strings: *"text" @str
  • Booleans: Result of comparisons

Functions

Functions in BombLang are defined using the tilde (~) symbol and can take parameters:

Function Definition

// Simple function without parameters
~sayHello_!alert "Hello from BombLang!"_

// Function with parameters
~double-n_*n & * & 2 @result !return result_

// Function with multiple parameters
~add-a-b_*a & + & b @sum !return sum_

// Calling functions
!sayHello
*5 @num
!double num @doubled
!alert "Doubled:" doubled

Function Return Values

Functions can return values using the !return statement:

~multiply-x-y_
  *x & * & y @product
  !return product
_

*6 @a
*7 @b
!multiply a b @result
!alert "6 * 7 =" result

Conditionals

BombLang supports if-else statements for conditional execution:

If-Else Statements

// Simple if statement
*age @userAge
*userAge & >= & 18:if_
  !alert "You are an adult!"
_

// If-else statement
*score @testScore
*testScore & >= & 60:if_
  !alert "You passed!"
_:else_
  !alert "You failed!"
_

// Nested conditionals
*temperature @temp
*temp & > & 30:if_
  !alert "It's hot!"
_:else_
  *temp & < & 10:if_
    !alert "It's cold!"
  _:else_
    !alert "It's mild!"
  _
_

Error Handling

BombLang provides try-catch blocks for error handling using the ^ symbol:

Error Handling

// Function that might throw an error
~safeDivide-a-b_
  *b & == & 0:if_
    !explode "divisionError" "Cannot divide by zero!"
  _
  *a & / & b @result
  !return result
_

// Try-catch block
*10 @numerator
*0 @denominator

^divisionError_
  !safeDivide numerator denominator @result
  !alert "Result:" result
_^divisionError_
  !alert "Error caught: Division by zero!"
_

Examples

Factorial Function

~factorial-n_
  *n & <= & 1:if_
    !return 1
  _
  *n & - & 1 @prev
  !factorial prev @prevFact
  *n & * & prevFact @result
  !return result
_

*5 @number
!factorial number @fact
!alert "Factorial of 5 is" fact

Fibonacci Sequence

~fibonacci-n_
  *n & <= & 1:if_
    !return n
  _
  *n & - & 1 @n1
  *n & - & 2 @n2
  !fibonacci n1 @fib1
  !fibonacci n2 @fib2
  *fib1 & + & fib2 @result
  !return result
_

*8 @fibNumber
!fibonacci fibNumber @fibResult
!alert "Fibonacci of 8 is" fibResult

Input Validation

~validateAge-age_
  *age & < & 0:if_
    !alert "Age cannot be negative"
    !return 0
  _
  *age & > & 150:if_
    !alert "Age seems unrealistic"
    !return 0
  _
  !alert "Valid age:" age
  !return 1
_

*25 @userAge
!validateAge userAge @isValid

Advanced Features

Chained Operations

You can chain multiple operations together:

*5 @x
*x & + & 3 & * & 2 & - & 1 @result
!alert "Result:" result  // Result: 15

Function Composition

Functions can call other functions to create complex behavior:

~double-n_*n & * & 2 @doubled !return doubled_
~quadruple-n_!double n @temp !double temp @quad !return quad_

*5 @num
!quadruple num @result
!alert "Quadruple of 5 is" result

Recursive Functions

BombLang supports recursion for complex algorithms:

~countdown-n_
